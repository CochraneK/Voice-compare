# 部署 & 真机联调指南（医生友好版）

> 第一版代码已写完并本地跑通。这份指南解决三件事：**后端放哪、小程序怎么连上、手机怎么测**。全程不用你会写代码，照着点就行。

---

## 一、后端放哪？三个方案对比

| 方案 | 费用 | 域名是否已备案 | 上手难度 | 推荐度 |
|---|---|---|---|---|
| **微信云托管** | 有免费额度，之后按量（很低） | ✅ 自动给已备案域名 | ⭐ 最简单 | ⭐⭐⭐⭐⭐ |
| **腾讯云 CNB** (cnb.cool) | 免费额度大 | ✅ 给 `*.cnb.run` HTTPS 域名 | ⭐⭐ 稍配置 | ⭐⭐⭐⭐ |
| GitHub Pages / Vercel | 免费 | ❌ 不能跑 Python 后端 | — | ❌ 不适用 |

**结论：用微信云托管。** 原因：它给的小程序后端域名是**已备案的 HTTPS**，能直接加进小程序白名单，免去你自己买域名+备案 1~2 周的麻烦。与小程序同属微信生态，控制台一键关联。

> GitHub 只能放静态网页，跑不了 Python 声纹分析，**不要选**。海外平台（Render/Railway）域名虽 HTTPS 但国内访问慢、且仍需在小程序白名单登记，无优势。

## 二、mp3 解码依赖：已帮你改好

原方案小程序录 mp3，服务端要装 ffmpeg。**我已把录音格式改成 wav**（`pages/record/record.js` 的 `format: 'wav'`）。好处：
- librosa 读 wav **不需要 ffmpeg**，只需 `libsndfile`（Dockerfile 里已装好）
- 部署到云托管/CNB 零额外系统依赖，最省心
- 代价只是文件稍大（10 秒录音约 170KB），上传无感，且音质更好（医学场景优先）

所以你**不用再管 ffmpeg**，直接用下面的 Dockerfile 部署即可。

## 三、方案 A：微信云托管部署（推荐）

### 准备：先注册一个小程序账号
1. 打开 https://mp.weixin.qq.com/
2. 点「立即注册」→ 选「小程序」→ 主体选「个人」（用你身份证，免费）
3. 注册完拿到 **AppID**（在「开发管理 → 开发设置」里），记下来

### 部署后端
1. 打开 **微信云托管**：https://cloud.weixin.qq.com/ （用上面注册的微信扫码登录）
2. 点「立即使用」→ 关联你刚注册的小程序
3. 进入控制台 →「服务列表」→「新建服务」
   - 服务名：`voice-compare`
   - 来源：**本地代码** 或 **代码仓库**（Dockerfile 在仓库根目录，直接上传整个仓库即可）
4. 部署设置：
   - 监听端口填 `9000`（和 Dockerfile 里一致）
   - 上传方式：选「文件夹上传」，把整个项目仓库传上去（含根目录的 Dockerfile）
5. 点「发布」等 1~2 分钟，构建完成
6. 发布成功后，在服务详情里找到 **公网访问域名**，形如：
   `https://voice-compare-xxx.ap-shanghai.run.tcloudbase.com`
   👉 **这个域名就是你要填进小程序的 baseUrl**，已备案、HTTPS，可直接加白名单。

> 微信云托管对新用户有免费额度，你这个声纹比对应用调用量很小，基本不会产生费用。

## 四、方案 B：腾讯云 CNB（免费备选）

如果想完全免费、且不关联小程序：

1. 打开 https://cnb.cool/ → 注册账号（微信扫码即可）
2. 新建组织 → 新建仓库 → 把整个项目仓库推上去（或用 WebIDE 上传）
3. 在 WebIDE 终端确认服务能起：`uvicorn main:app --host 0.0.0.0 --port 9000`
4. 在底部 **PORTS 面板** 添加 9000 端口 → CNB 会生成一个公网 URL，形如：
   `https://xxxx-9000.cnb.run`
5. 这个 HTTPS 域名同样可加进小程序白名单。

> CNB 免费额度大，但**开发环境会休眠**（一段时间不访问会停），不适合长期稳定服务。做演示和调试够用，正式用建议转方案 A。

## 五、小程序侧配置（连上后端）

### 1. 改 baseUrl
打开 `miniprogram/app.js`，找到：
```js
globalData: {
  baseUrl: 'http://127.0.0.1:8000'  // ← 改成你的云托管域名
}
```
改成你拿到的域名，例如：
```js
baseUrl: 'https://voice-compare-xxx.ap-shanghai.run.tcloudbase.com'
```
（注意：**不要**带末尾斜杠 `/`，因为 api.js 里 url 是 `/api/analyze-step`）

### 2. 加域名白名单（关键，不加真机必失败）
1. 登录 https://mp.weixin.qq.com/ → 进入你的小程序
2. 「开发管理」→「开发设置」→「服务器域名」→ 点「修改」
3. 在 **uploadFile 合法域名** 和 **request 合法域名** 两栏，都加上你的云托管域名：
   `https://voice-compare-xxx.ap-shanghai.run.tcloudbase.com`
4. 保存（每月可改 50 次，随意改）

## 六、真机联调：要下载什么、怎么做

### 需要下载的东西（就一个）
**微信开发者工具**（必装，官方免费）：
- 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 选「稳定版 Stable Build」，Windows 64 位版
- 安装后用注册小程序的微信扫码登录

### 不需要下载的
- 不用装 Python、不用装 Node（后端已在云端跑）
- 不用装模拟器（开发者工具自带模拟器 + 真机预览）

### 联调步骤
1. **导入项目**
   - 打开微信开发者工具 → 「+」导入项目
   - 目录选：`C:\Users\cunyi\WorkBuddy\2026-07-19-15-04-38\miniprogram`
   - AppID 填你注册的小程序 AppID（不要用测试号，测试号无法真机预览含录音权限的功能）
   - 点「导入」

2. **本地模拟器先验证**
   - 左侧模拟器会显示首页
   - 点「咨询师」→ 录音（模拟器里录音用的是电脑麦克风）→ 看是否拿到 token
   - 如果报「url not in domain list」：在开发者工具右上角「详情」→「本地设置」→ 勾选「**不校验合法域名**」（仅开发时用，真机必须配白名单）

3. **真机预览**
   - 顶部点「预览」→ 生成二维码
   - 用**手机微信**扫这个码 → 小程序就在手机上打开了
   - 在手机上走完整流程：录咨询师 → 录来访者 → 看报告

4. **真机调试**（看日志）
   - 顶部点「真机调试」→ 生成另一个码
   - 手机微信扫码 → 会建立调试通道
   - 电脑上能看到手机运行的 `console.log` 和报错，方便排查

### 常见报错对照
| 报错 | 原因 | 解决 |
|---|---|---|
| `url not in domain list` | 域名没加白名单 | 见第五步第 2 点，去公众平台加 |
| `request:fail ssl hand shake error` | 域名不是 HTTPS | 必须用云托管/CNB 给的 https 域名 |
| 录音没反应 | 没授权 scope.record | 首次会弹授权，或在手机设置里开 |
| 上传后 500 | 后端没起来/端口不对 | 看云托管日志，确认端口 9000、Dockerfile 正常 |

## 七、最短上线路径（TL;DR）

```
1. mp.weixin.qq.com 注册个人小程序 → 拿 AppID
2. cloud.weixin.qq.com 开通云托管 → 上传整个项目 → 拿 HTTPS 域名
3. miniprogram/app.js 改 baseUrl = 你的域名
4. 公众平台「服务器域名」加白名单（uploadFile + request 两栏）
5. 下微信开发者工具 → 导入 miniprogram → 预览 → 手机扫码测
```

全程预计 30~60 分钟（不含等云托管构建的几分钟）。卡住任何一步告诉我，我帮你查。
